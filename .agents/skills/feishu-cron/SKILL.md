# 飞书定时任务

## 强制规则

只要用户在说提醒、定时、稍后、延迟、每天几点、每周几、某个时间执行，你都必须调用 `cron_task`，不能只回复“好的，到时候提醒你”。

你没有后台线程。真正能把任务记住并在之后触发的，只有 `cron_task`。

## 什么时候用

- “20 分钟后提醒我开会”
- “每天早上 9 点发日报”
- “每周一 10 点提醒我交周报”
- “看看我现在有哪些提醒”
- “把那个提醒删掉”
- “现在立刻跑一次这个定时任务”

## 工具用法

### 新增任务

调用 `cron_task`：

```json
{
  "action": "add",
  "name": "可选，任务名",
  "time": "20m / 2026-04-16T13:30:00+08:00 / 0 9 * * *",
  "tz": "Asia/Shanghai",
  "prompt": "到时真正要执行的提示词"
}
```

规则：

- `time` 是相对时间时，会创建一次性任务。
- `time` 是 cron 表达式时，会创建周期任务。
- 用户没明确给时区时，默认走网关时区。
- `prompt` 写成到时要真正执行的话，不要写“用户刚刚说”这类对话上下文描述。

### 查看任务

```json
{ "action": "list" }
```

### 更新任务

先 `list` 拿到 `job_id`，再调用：

```json
{
  "action": "update",
  "job_id": "cron_xxx",
  "name": "可选，新任务名",
  "time": "可选，20m / 2026-04-16T13:30:00+08:00 / 0 9 * * *",
  "tz": "可选，Asia/Shanghai",
  "prompt": "可选，新的执行提示词"
}
```

规则：

- `job_id` 必填。
- `name`、`time`、`prompt` 至少传一个；只传 `tz` 不算更新。
- `time` 的规则和新增任务相同。
- 更新暂停中的任务不会自动恢复。

### 暂停任务

先 `list` 拿到 `job_id`，再调用：

```json
{ "action": "pause", "job_id": "cron_xxx" }
```

规则：

- 暂停只是不再自动执行，任务仍保留。
- 暂停不影响正在执行中的那一次。

### 恢复任务

先 `list` 拿到 `job_id`，再调用：

```json
{ "action": "resume", "job_id": "cron_xxx" }
```

### 恢复全部

```json
{ "action": "resume_all" }
```

规则：

- 只恢复当前会话范围里的已暂停任务。
- 私聊只处理当前用户的任务，群聊只处理当前群的任务。

### 删除任务

先 `list` 拿到 `job_id`，再调用：

```json
{ "action": "remove", "job_id": "cron_xxx" }
```

### 立即执行

先 `list` 拿到 `job_id`，再调用：

```json
{ "action": "run", "job_id": "cron_xxx" }
```

规则：

- `action=run` 不是当场拿到执行结果，而是把这次执行安排到当前回复结束后开始。
- 这次执行会像普通 cron 一样单独给用户发消息，不要在当前回复里伪造执行结果。
- 当前回复里只说明“已安排稍后执行”，不要写“我刚刚已经替你跑完了”。
- `run` 只额外执行一次，不改变任务的暂停或恢复状态。

## 回复原则

- 工具成功后，再用自然语言告诉用户结果。
- `action=run` 成功后，只说已经安排在当前回复结束后执行，结果会稍后单独发给用户。
- 如果用户时间说得不清楚，先追问，不要瞎猜。
- 如果用户只是想“定时跑一个任务”，`prompt` 里写清楚最终要做什么，不要只写一句空话。
