# 飞书定时任务

## 强制规则

只要用户在说提醒、定时、稍后、延迟、每天几点、每周几、某个时间执行，你都必须调用 `cron_task`，不能只回复“好的，到时候提醒你”。

你没有后台线程。真正能把任务记住并在之后触发的，只有 `cron_task`。

## 什么时候用

- “20 分钟后提醒我开会”
- “每天早上 9 点发日报”
- “每周一 10 点提醒我交周报”
- “看看我现在有哪些提醒”
- “把那个提醒删掉”
- “现在立刻跑一次这个定时任务”

## 工具用法

### 新增任务

调用 `cron_task`：

```json
{
  "action": "add",
  "name": "可选，任务名",
  "time": "20m / 2026-04-16T13:30:00+08:00 / 0 9 * * *",
  "tz": "Asia/Shanghai",
  "prompt": "到时真正要执行的提示词"
}
```

规则：

- `time` 是相对时间时，会创建一次性任务。
- `time` 是 cron 表达式时，会创建周期任务。
- 用户没明确给时区时，默认走网关时区。
- `prompt` 写成到时要真正执行的话，不要写“用户刚刚说”这类对话上下文描述。

### 查看任务

```json
{ "action": "list" }
```

### 删除任务

先 `list` 拿到 `job_id`，再调用：

```json
{ "action": "remove", "job_id": "cron_xxx" }
```

### 立即执行

先 `list` 拿到 `job_id`，再调用：

```json
{ "action": "run", "job_id": "cron_xxx" }
```

## 回复原则

- 工具成功后，再用自然语言告诉用户结果。
- 如果用户时间说得不清楚，先追问，不要瞎猜。
- 如果用户只是想“定时跑一个任务”，`prompt` 里写清楚最终要做什么，不要只写一句空话。
