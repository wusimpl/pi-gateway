# Pi Gateway 后台管理模块 PRD

## 1. 背景

Pi Gateway 现在主要通过飞书里的 `/命令` 管理会话、模型、工具、定时任务和群聊设置。

随着命令变多，在飞书里手动输入不够方便。尤其电脑就在身边时，用户更希望打开网页，通过按钮、下拉框和输入框快速完成同样操作。

## 2. 产品目标

做一个轻量后台网页，作为飞书 `/命令` 的快捷操作替代。

核心目标：

- 少打字
- 不改变现有命令能力
- 私聊和群聊可以分开操作
- 所有命令都能找到入口
- 页面保持简单，不做复杂后台系统

## 3. 非目标

第一版不做：

- 不做总览大盘
- 不做复杂数据统计
- 不做用户管理系统
- 不做定时任务新建表单
- 不做聊天记录管理
- 不做飞书通讯录全量管理
- 不做公网后台能力

## 4. 使用对象

主要使用者是网关 owner。

典型场景：

- 在电脑旁快速切换模型
- 查看某个私聊或群聊的会话状态
- 停止当前正在执行的任务
- 查看某个群的定时任务并暂停或恢复
- 查看或调整群聊触发设置
- 管理当前目标里的工具开关

## 5. 核心设计原则

### 5.1 先选目标，再执行命令

后台顶部始终有一个“执行目标”选择器。

目标类型：

- 私聊
- 群聊
- 手动输入 ID

选定目标后，页面里的所有操作都作用在这个目标上。

### 5.2 私聊和群聊必须隔离

同一个命令，在不同目标里结果可能不同。

例如：

- `/sessions`：不同私聊、不同群聊的会话列表不同
- `/cron list`：不同目标的定时任务不同
- `/group`：只对当前群聊有意义
- `/tools`：不同会话的工具状态可能不同

后台必须明确显示当前目标，避免误操作。

### 5.3 保留原始命令入口

每个功能页可以有表单或按钮，但必须保留一个“原始命令输入框”。

这样即使某个命令还没有专门界面，也可以直接执行：

```text
/model
/cron list
/group keywords show
/tools reset
```

## 6. 页面范围

第一版只需要以下页面。

### 6.1 登录页

用途：防止本机后台被随便访问。

内容：

- 管理密码输入框
- 登录按钮

第一版只需要简单密码即可。

### 6.2 执行目标页

用途：选择当前要操作的私聊或群聊。

功能：

- 查看已知私聊
- 查看已知群聊
- 搜索 ID 或名称
- 手动输入 open_id 或 chat_id
- 切换当前执行目标

目标来源：

- 历史私聊
- 历史群聊
- owner 配置
- 群白名单
- 定时任务里出现过的目标
- 手动添加

### 6.3 全部命令页

用途：兜住所有 `/命令`。

功能：

- 命令分类展示
- 点击命令后显示简单参数输入
- 可直接输入完整命令
- 执行后显示结果
- 可选是否把结果发回飞书

命令分类：

- 会话：`/new` `/reset` `/status` `/sessions` `/resume` `/context`
- 模型：`/models` `/model` `/route`
- 设置：`/settings` `/stream` `/stt` `/reaction` `/toolcalls` `/skill-folder`
- 工具：`/tools`
- 定时任务：`/cron`
- 控制：`/stop` `/next` `/restart`
- 群聊：`/group`
- 技能：`/skills` `/skillstat`

### 6.4 会话页

用途：管理当前目标的会话。

功能：

- 查看当前状态
- 新建会话
- 重置会话
- 查看历史会话
- 恢复历史会话
- 查看已加载上下文

对应命令：

```text
/status
/new
/reset
/sessions
/resume <序号或会话ID>
/context
```

### 6.5 模型页

用途：查看和切换当前目标的模型设置。

功能：

- 查看当前模型
- 查看可用模型
- 设置 router / light / heavy 模型
- 开关模型路由

对应命令：

```text
/model
/models
/model router <模型>
/model light <模型>
/model heavy <模型>
/route on
/route off
```

### 6.6 定时任务页

用途：查看和控制当前目标的定时任务。

第一版只做：

- 查看任务列表
- 立即运行任务
- 停止正在运行的任务
- 删除任务

不做：

- 不做新建任务
- 不做复杂编辑表单

对应命令：

```text
/cron list
/cron run <jobId>
/cron stop <jobId>
/cron remove <jobId>
```

### 6.7 群聊设置页

用途：管理当前群聊的触发规则。

只在目标是群聊时显示。

功能：

- 查看群聊设置
- 设置群聊开关
- 设置触发方式
- 查看关键词
- 设置或清空关键词
- 查看白名单
- 当前群加入或移出白名单

对应命令：

```text
/group
/group policy disabled|allowlist|open
/group mode mention|all|keyword
/group keywords show
/group keywords set <关键词>
/group keywords clear
/group allowlist show
/group allowlist add here
/group allowlist remove here
```

### 6.8 工具页

用途：管理当前目标会话里的工具开关。

功能：

- 查看工具列表
- 启用工具
- 禁用工具
- 设置为指定工具集合
- 恢复默认

对应命令：

```text
/tools
/tools on <tool>
/tools off <tool>
/tools set <tool>
/tools reset
```

## 7. 关键交互

### 7.1 执行命令

流程：

1. 用户选择执行目标
2. 用户选择页面或命令
3. 用户点击执行
4. 页面显示执行结果
5. 如用户勾选“同步到飞书”，结果也发回对应私聊或群聊

### 7.2 未选择目标

如果没有选择目标，不允许执行命令。

页面提示：

```text
请先选择私聊或群聊。
```

### 7.3 群聊命令限制

当目标不是群聊时，不显示群聊设置页。

如果用户强行执行 `/group`，提示：

```text
请先选择一个群聊。
```

### 7.4 未知目标

如果目标不在已知列表里，允许手动输入 ID。

手动目标需要用户选择类型：

- 私聊 open_id
- 群聊 chat_id

## 8. 第一版验收标准

第一版完成后，应满足：

- 可以登录后台
- 可以选择私聊或群聊
- 可以手动输入目标 ID
- 可以执行所有已有 `/命令`
- 会话、模型、定时任务、群聊设置、工具页可用
- 定时任务页支持查看、运行、停止、删除
- 不提供定时任务新建入口
- 不提供总览页
- 执行结果能在页面显示
- 可选同步结果到飞书
