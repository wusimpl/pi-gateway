# Pi Gateway

Pi Gateway 是一个把 **Pi 接到飞书**里的助手。

你可以像跟同事聊天一样，在飞书私聊或群聊中直接让它帮你写东西、查资料、处理文件、整理语音、生成文档、设置定时任务，或者继续完成更复杂的工作。

它的目标很简单：

> 让你不用离开飞书，就能使用 Pi 的能力。

---

## 它能做什么

你可以直接把需求发给机器人，例如：

- 写文案、改稿、总结、翻译、润色
- 分析网页、代码、报错、日志、需求文档
- 读取图片、识别截图内容
- 转写语音，并基于语音内容继续处理
- 接收文件，并根据文件内容继续工作
- 创建、读取、追加、替换飞书文档
- 把生成好的文件直接发回飞书
- 设置一次性或周期性的定时任务
- 在群聊中协作使用同一个上下文
- 管理会话、模型、工具和回复方式

你不需要记住复杂规则。多数情况下，直接说你想做什么就可以。

---

## 私聊怎么用

在飞书里打开和机器人的私聊，直接发送消息即可。

你可以发送：

- **文字**：直接提问或下指令
- **图片**：截图、照片、图表、报错界面等
- **语音**：机器人会先转写，再根据内容处理
- **文件**：机器人会保存文件，并根据你的要求读取或处理
- **引用消息**：引用之前的消息继续追问，机器人会带上引用内容

示例：

```text
帮我把这段话改得更适合发朋友圈
```

```text
总结一下这个文件里的重点，给我列成待办
```

```text
这张截图里报错是什么意思？怎么解决？
```

---

## 群聊怎么用

在群聊里使用时，通常需要 **@ 机器人** 后再提问。  
如果你把群消息接收模式配置成关键词触发，也可以直接发送带关键词的消息。

示例：

```text
@Pi 帮我们总结一下刚才讨论的结论
```

群聊里的会话和私聊里的会话是分开的。一个群聊会有自己的上下文，不会默认混进你的私聊上下文。

群聊中部分管理命令可能只有 owner 可以使用，避免普通成员误删任务、切换会话或重启网关。

---

## 常用命令

命令都以 `/` 开头。

| 命令 | 用途 |
| --- | --- |
| `/new` | 新建一个会话 |
| `/reset` | 重置当前会话 |
| `/status` | 查看当前会话状态 |
| `/sessions` | 查看历史会话 |
| `/resume <序号或会话ID前缀>` | 恢复某个历史会话 |
| `/stop` | 停止当前正在执行的任务 |
| `/next <内容>` | 把补充内容排到当前任务后处理 |
| `/models` | 查看当前可用模型 |
| `/model` | 查看当前模型 |
| `/model <序号或模型名>` | 切换模型 |
| `/settings` | 查看当前设置 |
| `/tools` | 查看或管理工具 |
| `/toolcalls` | 设置是否展示工具调用 |
| `/cron` | 查看定时任务用法 |
| `/cron list` | 查看当前定时任务 |
| `/skills` | 查看当前可用技能 |
| `/skillstat` | 查看技能使用统计 |
| `/stt provider <名称>` | 切换语音转写方式 |
| `/stream on/off` | 开关流式回复 |
| `/reaction on/off` | 开关处理中表情 |
| `/restart` | 重启网关 |

## 会话管理

Pi Gateway 会保留你的会话。你可以随时新建、重置或恢复历史会话。

### 新建会话

```text
/new
```

适合开始一个全新的主题。

### 重置当前会话

```text
/reset
```

适合当前对话已经跑偏，想清空上下文重新来。

### 查看历史会话

```text
/sessions
```

如果会话很多，可以翻页：

```text
/sessions -n 2
```

### 恢复历史会话

```text
/resume 3
```

也可以用会话 ID 前缀恢复：

```text
/resume abc123
```

## 模型和回复设置

### 查看可用模型

```text
/models
```

按来源筛选：

```text
/models openai
```

### 切换模型

```text
/model 2
```

或：

```text
/model provider/model-name
```

### 调整推理强度

```text
/settings think medium
```

可选值：

```text
off, minimal, low, medium, high, xhigh
```

### 开关流式回复

```text
/settings stream on
```

或：

```text
/settings stream off
```

## 工具管理

查看当前可用工具：

```text
/tools
```

启用某些工具：

```text
/tools on read bash
```

禁用某些工具：

```text
/tools off bash
```

只保留指定工具：

```text
/tools set read write
```

恢复默认工具：

```text
/tools reset
```

---

## 工具调用展示

如果你想知道机器人执行过程中用了哪些工具，可以开启工具调用展示。

查看当前模式：

```text
/toolcalls
```

只显示工具名：

```text
/toolcalls name
```

显示详情：

```text
/toolcalls full
```

关闭展示：

```text
/toolcalls off
```

## 定时任务

Pi Gateway 可以帮你创建提醒、日报、定时检查、周期总结等任务。

### 查看定时任务

```text
/cron list
```

### 新建一次性任务

```text
/cron add
name: 20分钟后提醒
at: 20m
prompt:
提醒我去参加会议，只输出一句简短提醒。
```

### 新建周期任务

```text
/cron add
name: 每日天气播报
cron: 0 7 * * *
tz: Asia/Shanghai
prompt:
查询珠海当天天气，并用简洁中文播报。
```

### 立即执行任务

```text
/cron run <jobId>
```

如果当前会话还有任务在跑，立即执行会被安排到当前回复结束后再开始。

### 删除任务

```text
/cron remove <jobId>
```

注意：删除前建议先用 `/cron list` 确认任务 ID。

## 图片、语音和文件

### 图片

你可以直接发送截图、照片、图表、报错界面。

机器人会尽量直接理解图片内容；如果当前模型不支持看图，会使用识别结果继续处理。

### 语音

你可以直接发送飞书语音。

机器人会先转写成文字，再根据转写内容继续执行你的需求。

切换语音转写方式：

```text
/stt provider sensevoice
```

也可以切换为：

```text
/stt provider whisper
```

或：

```text
/stt provider doubao
```

### 文件

你可以直接发送文件，并告诉机器人要怎么处理。

例如：

```text
帮我总结这个 PDF 的重点
```

```text
把这个表格整理成适合发给老板的版本
```

生成好的最终文件，也可以让机器人直接发回飞书。

## 飞书文档

Pi Gateway 可以操作飞书新版文档。

常见用法：

- 新建一篇飞书文档
- 读取已有文档内容
- 往文档末尾追加内容
- 替换整篇文档正文
- 删除文档中的部分内容
- 删除整篇文档

删除整篇文档属于高风险操作，机器人会要求你明确确认后才会执行。

如果文档里包含真实可访问的图片地址，机器人会优先尝试把图片直接嵌入文档；如果权限、链接或下载失败，才会退回为图片链接。

## 运行中的任务

当机器人正在处理任务时，你可能会看到：

- 消息上出现处理中表情
- 回复以流式卡片形式逐步更新
- 长回复被自动拆成多条消息
- 工具调用过程被展示出来

如果想停止当前任务：

```text
/stop
```

如果想补充一句，但不想打断当前任务：

```text
/next 另外请把结果整理成表格
```

## 私聊、群聊和上下文

Pi Gateway 会尽量按会话隔离上下文：

- 私聊有私聊自己的上下文
- 每个群聊有自己的上下文
- 不同群聊之间不共享上下文
- 群聊任务不应该出现在私聊任务列表中
- 私聊任务不应该出现在群聊任务列表中

如果你发现任务、会话或回复内容串到了不该出现的地方，请立即反馈。这类问题会被视为作用域隔离问题处理。

## 常见问题

### 机器人没有回复怎么办？

可以先检查：

- 私聊里是否直接发给了机器人
- 群聊里是否 @ 了机器人
- 当前是否已有任务正在执行
- 是否需要用 `/stop` 停掉卡住的任务

### 群聊里为什么没反应？

群聊可能只响应 @ 机器人的消息，或者只响应带关键词的消息。请尝试：

```text
@Pi 帮我看看这个问题
```

如果仍然没反应，可能是当前群聊没有启用机器人。

### 怎么停止正在跑的任务？

发送：

```text
/stop
```

### 怎么换模型？

先看可用模型：

```text
/models
```

再切换：

```text
/model 2
```

### 怎么找回之前的对话？

先查看历史会话：

```text
/sessions
```

再恢复：

```text
/resume 1
```

### 为什么语音识别失败？

可能是语音太长、声音太小、转写服务不可用，或当前转写方式没有配置好。可以换一种转写方式再试。

### 为什么飞书文档里的图片没有嵌进去？

通常是图片地址不可直接访问、需要登录、下载失败，或机器人没有足够的图片上传权限。遇到这种情况，机器人会尽量保留图片链接。

---

## 使用建议

- 复杂任务尽量说清楚目标、格式和限制
- 长任务运行时，用 `/next` 补充信息，不要连续刷屏
- 删除任务、删除文档、重启网关前先确认影响范围
- 群聊里不要发送不适合被所有成员看到的敏感内容

Pi Gateway 的定位不是另一个复杂系统，而是一个在飞书里随叫随到的 Pi 助手。直接说需求，让它干活就好。
